# Contributing to Raze

Thanks for your interest in contributing. Raze is in early development and all contributions are welcome.

## Ways to contribute

- **Report bugs** — open an issue with steps to reproduce
- **Test on your contracts** — try Raze on your Foundry projects and share what you find
- **Suggest vulnerability classes** — currently covers reentrancy, access control, arithmetic, flash loan, price manipulation
- **Improve documentation** — fix typos, clarify steps, add examples
- **Submit PRs** — for bug fixes or small improvements, open a PR directly. For larger changes, open an issue first to discuss

## Getting started

1. Fork the repo
2. Clone your fork
3. Run `raze doctor` to verify your environment
4. Make your changes
5. Open a PR with a clear description of what you changed and why

## Code style

TypeScript + Zod v4. Keep it consistent with the existing codebase.

## Questions

Open an issue or start a discussion. Building in public from day one — no question is too small.
